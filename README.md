 Vulnerability Management & SOC Lab – Ubuntu + Nessus + Qualys

This repository documents my hands-on journey into vulnerability management and SOC (Security Operations Center) work. \
The goal of this project is to demonstrate real world security analyst skills by performing full vulnerability scanning, remediation, and verification cycles using enterprise-grade tools.

---

 Environment & Tools

- Ubuntu (VirtualBox)
- Nessus Essentials – vulnerability scanning
- Qualys VMDR – agent-based scanning
- Git + GitHub (SSH) – secure version control
- OpenSSH server – for authenticated scans
- Linux CLI – for patching and remediation

---

  Skills Practiced

- Running unauthenticated and authenticated scans 
- Reading and understanding vulnerability findings 
- Interpreting CVSS scores and risk levels 
- Developing remediation and patching plans 
- Running a verification scan after fixes 
- Documenting findings and writing professional reports 
- Working with Linux commands and system administration 
- Building a real cybersecurity portfolio

---

 Repository Structure

- lab/scan01 – Baseline unauthenticated Nessus scan 
- lab/scan02 – Deep authenticated Nessus scan (SSH credentials) 
- lab/scan03 – Post-remediation verification scan 
- reports/ – Executive summaries and remediation plans 
- analysis-cve/ – CVE writeups, CVSS scoring, and reasoning 
- docs/ – Lab notes, checklists, and setup documentation 
- assets/ – Screenshots and evidence used in reports 

---

 Project Goal

build a complete, realistic vulnerability management workflow,the same way and process used by SOC analysts and vulnerability analysts in real organizations.
